HTML

  • HTML stands for Hypertext Markup Language
  • Creates the structure of a webpage
  • An element is a type of content on a webpage
  • head element contains information about the webpage
  • body element represents the visible content shown to the user
  • title element defines the title for a webpage
  • style element defines the CSS styles associated with a website
  • base element defintes the base URL for a webpage
  • link element connects an external resource to the HTML document
  • meta element defines the metadata such as the character set, description, keywords, author, and viewport
  • script element embeds client-side scripts in the HTML document
  • ul element creates an unordered lists
  • li element creates a list item that can go inside of the ul element
  • The names of semantic elements describe its contents and should be used over non-semantic elements when possible
  • Examples of semantic elements are section
  • Examples of non-semantic elements are div and span

CSS

  • CSS stands for Cascading Style Sheets
  • You can add CSS to your webpage with inline CSS, internal style sheets, or external style sheets
  • A CSS rule sets styles for elements and is made up of a selector, declaration block, and declarations
  • A declaration block is the section of code starting and ending with curly brackets that contains declarations
  • A declaration contains a property and value (e.g. color: blue;)
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element

Git

  • Git is a version control system that allows you to save versions of files and makes collaboration easier
  • git config --global user.name "[firstname lastname]": allows you to configure user name across local repositories
  • git config --global user.email "[email]": allows you to configure user email across local repositories
  • git clone [url]: retrieves an entire repository from a hosted location via URL
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git status: shows current branch and shows file info such as whether files are staged, unstaged, tracked or untracked
  • git add -A: Adds all modifications to staging area
  • git commit -m "[message]": creates a snapshot of all your staged files
  • git pull origin [branch]: Grabs any changes from the remote repository's specified branch
  • git push origin [branch]: Sends any changes to a remote repository's specified branch

JavaScript

  • Used to add dynamic interactivity to websites such as button functionality or pop up windows
  • You can use script tags to add JavaScript to your website
  • Code runs from top to bottom but you can use conditional statements and loops to control the flow.
  • Case-sensitive
  • Ignores whitespace
  • Indentation is for readability
  • // This is a single line comment in Javascript
  • /* This is also a comment but it can be multiple lines */
  • Semicolons are placed at the end of statements - only required when there are multiple statements in one line
  • There are many different data types such as strings, numbers, symbols, and booleans
  • An operator is a mathematical symbol (e.g. =, +, -, *, /, etc) that produces a result based on two values or variables
  • Variables are containers to store data and can be created using the var keyword
  • Functions contain code that can be reused and can be created using the function keyword
  • Functions must be called to be used
